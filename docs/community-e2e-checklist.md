# ScoreLink 커뮤니티 수동 시나리오 체크리스트

`npm run build && npx vite preview`로 dist를 서브하거나 배포된 환경에서 점검한다.
Supabase 프로젝트가 `.env.local`(`VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY`)로 연결되어
있어야 하며, "타인" 시나리오 검증을 위해 서로 다른 Google 계정 2개 이상이 필요하다.

## 1. 로그인/계정

- [ ] 헤더 "Google 로그인" → OAuth 동의 화면 → 복귀 후 헤더가 로그인 상태(내 릭·로그아웃)로 전환
- [ ] 첫 로그인 시 `profiles` 행이 자동 생성됨 (표시 이름·아바타가 Google 메타데이터로 채워짐)
- [ ] 로그아웃 → 헤더가 비로그인 상태로 복귀, `/publish`·`/me` 접근 시 로그인 안내
- [ ] 새로고침 후에도 로그인 세션 유지

## 2. 게시·중복 판정

- [ ] 편집기에서 공유 링크(`?mode=view#v1.…`) 복사 → `/publish`에 붙여넣기 → 미리보기 렌더 → 게시 → `/lick/:id`로 이동
- [ ] `/publish#v1.…`로 직접 접속 시 해시가 입력창에 자동 채워짐
- [ ] 음표가 없는 빈 곡 링크 붙여넣기 → 게시 버튼 비활성 또는 게시 거부(오류 메시지)
- [ ] 방금 게시한 릭을 동일 계정으로 다시 게시 → "이미 게시한 릭" 안내(23505 unique violation, 재게시 차단)
- [ ] 같은 릭을 조옮김(피치만 이동)해서 재게시 → 멜로디 해시 동일 → 마찬가지로 재게시 차단
- [ ] 같은 릭을 템포/코드만 바꿔 재게시 → 역시 멜로디 동일 판정으로 차단
- [ ] 다른 계정으로 동일 멜로디를 게시 → 게시는 허용(유사릭)되고 원본과 별도 id로 등록됨

## 3. 좋아요

- [ ] 원본 릭에 좋아요 → 카운트 +1, 하트 활성화
- [ ] 유사릭 상세에서 좋아요 → 원본 릭의 카운트에 반영되고(귀속), 유사릭 자체 카운트는 증가하지 않음
- [ ] 같은 계정으로 원본과 유사릭 양쪽에서 좋아요를 눌러도 카운트는 1만 증가(이중 카운트 불가 — PK가 `(user_id, canonical_id ?? id)`)
- [ ] 비로그인 상태에서 좋아요 버튼 클릭 → "로그인 후 누를 수 있어요" 안내, 카운트 변화 없음
- [ ] 좋아요 버튼을 빠르게 더블클릭 → 요청 진행 중 버튼 비활성화(single-flight)로 카운트가 2 이상 튀지 않음
- [ ] 좋아요 취소 → 카운트 -1, 하트 비활성화

## 4. 목록

- [ ] 피드(`/`)가 최신순으로 표시되고, 스크롤 시 keyset 커서(`created_at` 기준)로 다음 페이지가 이어붙음(중복·누락 없음)
- [ ] 랭킹(`/ranking`)이 좋아요 수 내림차순으로 정렬되고, 스크롤 중 순위가 바뀌어도 offset 방식 dedupe로 같은 릭이 중복 표시되지 않음
- [ ] 유사릭 상세에 "이 릭은 유사릭이에요" 배지와 원본 링크가 표시되고, 표시되는 좋아요 수는 원본(canonical) 기준
- [ ] 원본 상세·목록의 좋아요 수와 그 유사릭들에 표시되는 좋아요 수가 항상 일치
- [ ] 유저 페이지(`/user/:public_id`)에 해당 사용자가 게시한 릭이 최신순으로 나열됨
- [ ] `/me`(내 릭)에 로그인한 사용자 본인의 게시물만 나열되고 삭제 UI가 보임
- [ ] `/lick/:id`, `/user/:public_id` 주소를 새 탭에 직접 붙여넣거나 새로고침 → SPA rewrite로 정상 렌더(404 아님)

## 5. 삭제·승계

- [ ] 유사릭(canonical_id 있음) 삭제 → 원본과 다른 유사릭들에는 영향 없음, 좋아요 수 변화 없음
- [ ] 다른 유사릭이 존재하는 원본을 삭제 → 가장 오래 전에 게시된 유사릭이 새 원본으로 승격되고, 기존 좋아요 전부와 나머지 유사릭들의 canonical_id가 승계됨
- [ ] (API 직접 호출로) 유사릭에 좋아요를 넣은 사용자가 원본에도 좋아요를 누른 상태에서 원본 삭제 → 삭제가 성공하고 승계자 좋아요는 1개만 남는다
- [ ] 유사릭이 하나도 없는 원본을 삭제 → 좋아요도 함께 사라짐(cascade), 오류 없이 완료
- [ ] 삭제는 본인 게시물에서만 버튼이 보이고, 확인 다이얼로그를 거친 뒤 실행됨

## 6. 재생

- [ ] 릭 상세에서 "전곡 재생" → 처음부터 재생, 버튼이 "정지"로 전환
- [ ] 재생 중 "정지" 클릭 → 즉시 무음, 버튼이 "전곡 재생"으로 복귀
- [ ] 악보의 음표를 탭 → 그 위치부터 멜로디만 재생(반주·메트로놈 없음)
- [ ] 재생 중 다른 릭 상세로 이동하거나 화면을 벗어나면 재생이 자동으로 정지됨

## 7. 레거시 호환

- [ ] 구버전 공유 링크(`?mode=view#v1.…`)를 그대로 열면 커뮤니티 라우팅과 무관하게 기존 열람 화면이 정상 렌더
- [ ] `/edit`로 진입한 편집기가 기존 편집 기능(입력·재생·공유)과 동일하게 동작
