# ScoreLink 기능 명세 (확정본)

프로토타입 반복을 통해 확정된 전체 사양. `prototype-reference.html`이 이 명세의 동작 레퍼런스이며, 명세와 프로토타입이 다르면 이 문서가 우선한다.

## 1. 제품 개요

악보 데이터를 압축해 URL 해시에 담아 서버 없이 공유하는 모바일 웹앱.
- **편집 화면**: 곡 작성. 크로매틱 드럼 패드 + 스테퍼 기반 입력.
- **열람 화면**: 공유 링크(`?mode=view`)로 열리는 감상용 화면.
- 백엔드 없음. 곡 전체가 URL에 인코딩됨.

## 2. 데이터 모델

```ts
interface Note {
  id: number;      // 세션 내 유니크
  s: number;       // 시작: 곡 처음부터의 절대 16분음표 인덱스 (pickup 포함)
  d: number;       // 길이: 16분음표 개수
  p: number;       // MIDI 피치 (53=파3 ~ 84=도6)
}
interface Song {
  title: string;
  tempo: number;               // 40~240, 기본 100
  meas: number;                // 일반 마디 수, 기본 4
  pickup: 0 | 8;               // 못갖춘마디 (0=없음, 8=2박)
  accPat: 'pad'|'comp'|'arp';  // 전역 반주 패턴, 기본 'pad'
  mAcc: Record<number, 'pad'|'comp'|'arp'|'off'>; // 마디별 오버라이드 (키=마디 인덱스)
  notes: Note[];
  chords: Record<number, string>; // 키 = 절대 박 인덱스 (s/4), 값 = "C","F♯m7" 등
}
```

핵심 원칙:
- **쉼표는 저장하지 않는다.** 음이 없는 구간은 렌더 시 자동 파생된다 (§5.4).
- 단선율(monophonic). 겹침은 입력/수정 시 자동 해소된다 (§7.1).
- 상수: `SPM=16`(마디당 16분음표), `PMIN=53`(파3), `PMAX=84`(도6),
  `LEN_STEPS=[1,2,3,4,6,8,12,16]`, `INPUT_LEN=4`(패드 입력 고정 길이).

### 2.1 마디 지오메트리 (pickup 지원)

pickup이 있으면 마디 인덱스 0이 못갖춘마디, 1..meas가 일반 마디.

```
TOTAL()        = pickup + meas*16
measCountAll() = meas + (pickup?1:0)
measStart(m)   = pickup ? (m===0 ? 0 : pickup+(m-1)*16) : m*16
measLen(m)     = (pickup && m===0) ? pickup : 16
measOf(step)   = pickup ? (step<pickup ? 0 : 1+floor((step-pickup)/16)) : floor(step/16)
lineOf(m)      = pickup ? (m===0 ? 0 : floor((m-1)/4)) : floor(m/4)   // MPL=4
measLabel(m)   = (pickup && m===0) ? "못갖춘" : `마디 ${pickup ? m : m+1}`
```

## 3. 화면 구성 — 편집

위에서 아래로: 헤더 / 메타라인 / 악보 / 코드 행 / (코드 피커) / 드럼 패드 / 마디 이동 바 / 스테퍼 / 6버튼줄.

### 3.1 헤더
- 좌: eyebrow "SCORELINK — EDIT" + 곡 제목.
- 우 (순서대로):
  1. **반주 버튼** (화음 아이콘): 탭 → 팝오버 [끔 | 패드 | 컴핑 | 아르페지오]. 패턴 선택=반주 켬+전역 패턴 설정, 끔=반주 끔. 켜짐 상태는 버튼 먹색 반전. 재생 중 변경 시 현재 위치부터 실시간 반영.
  2. **메트로놈 버튼**: 토글. 켜짐=먹색 반전. 재생 중 토글 시 실시간 반영(켜면 남은 박 스케줄, 끄면 클릭만 정지).
  3. **전체 재생/정지 버튼**: 곡 처음(step 0)부터 전체 재생. 재생 중 아이콘이 일시정지(⏸)로 바뀌고 다시 누르면 즉시 정지(소리 포함). 재생 중 현재 마디를 따라가며 curM 갱신 + 악보 줄 자동 전환.
  4. **템포 칩** "♩=100": 탭 → 팝오버 [− | 숫자 직접입력 | ＋ | 닫기]. ±4씩, 40~240 클램프.
  5. **공유 버튼**: `?mode=view#v1.…` 열람 링크를 클립보드 복사 + 현재 해시 갱신. clipboard 실패 시 prompt 폴백.
  6. **보기 버튼**: 열람 화면으로 전환.
- 메타라인: `C장조 · 4/4 · N마디( + 못갖춘)` / `자동 저장 · v1`.

### 3.2 악보 (표시 전용 + 마디 내비게이션)
- **한 줄(4마디)만 표시.** 현재 마디(curM)가 속한 줄을 렌더. 세로 스크롤 없음.
- 현재 마디: 레드 음영(opacity 0.055) + 좌우 레드 경계선.
- **마디 탭 = 그 마디로 이동 + 그 마디의 첫 노트 자동 선택**(마디에 걸친 음 포함, `n.s<me && n.s+n.d>ms` 중 s 최소). 노트 없으면 선택 해제. 선택 시 프리뷰 사운드. **음표 개별 탭/편집은 불가.**
- 못갖춘마디: 첫 줄에 일반 마디의 **70% 폭**(`PICKW=0.7`)으로 그려져 첫 줄이 4.5마디 상당이 됨. `mw = (W-여백)/(4 + (pickup?0.7:0))`.
- **콘텐츠 배치는 `posX`로 통일** (마디선 넘침 방지):
  ```
  posX(m, frac) = measX(m) + inset + frac*(measW(m) - inset - 6)   // inset = min(9, measW*0.18)
  ```
  음표·쉼표·코드·플레이헤드 모두 이 함수 사용.
- 음자리표·쉼표는 **SVG 패스 직접 드로잉** (SMP 유니코드 문자 사용 금지 — Android 폰트 부재 문제).
- 덧줄(ledger line): 보표 아래 step<0 → 짝수 step −2부터, 위 step>8 → 10부터.
- 기둥 방향: `step>=4`(시4 이상)이면 아래로, 미만이면 위로. 꼬리(flag)는 방향에 맞춰 미러.
- 임시표: C장조 스펠링, 검은건반은 ♯로 표기(레드 색).
- 마디 경계를 넘는 음: 분할 렌더 + 타이 아치(같은 줄 내에서만).
- 세로 상수: `LH=118`(줄 높이), `LINEGAP=10`, 보표 top=42(단일 줄 기준), 마디 히트영역 높이 98.

### 3.3 코드 행 (패드 바로 위)
- 좌측: **마디별 반주 버튼** ("반주" + 현재값). 탭마다 순환: 기본 → 패드 → 컴핑 → 아르프 → 끔 → 기본. "기본"=전역 패턴 따름. 끔이면 라벨 흐림.
- 우측: **박 단위 코드 슬롯** — 일반 마디 4개, 못갖춘마디 2개. 패드의 박 그룹과 시각 정렬.
- 슬롯: 코드 있으면 세리프 이탤릭 표기, 없으면 점선 테두리 + "＋".
- 슬롯 탭 → 코드 피커 개폐 (같은 슬롯 재탭=닫기).

### 3.4 코드 피커 (코드 행과 패드 사이에 펼쳐짐)
- 헤더: "마디 N · B박 코드" + 현재 코드. 못갖춘마디의 박 표기는 3, 4.
- 3단 그리드: 루트 [C D E F G A B] / 임시표 [♮ ♯ ♭] / 성질 [maj min 7 maj7 m7 dim aug sus4].
- **선택 즉시 적용** (별도 확인 없음). 하단 [코드 지우기 | 완료].
- 저장 문자열: `루트 + (♯|♭|"") + 성질접미` (maj=빈 문자열, min="m").

### 3.5 드럼 패드
- 구조: 박 헤더(상단 고정) / 음 행 스크롤 영역(높이 156px) / 쉼표 행(하단 고정).
- 그리드: 16분음표 단위. 4분음표(4칸) 그룹핑 — 그룹 간 여백 7px, 각 그룹 첫 칸(정박) 진한 음영, 박 헤더에 번호(일반 1~4, 못갖춘 3~4). 못갖춘마디는 8열(2그룹).
- **행 구성 (동적)**: 온음계 행은 파3~도6 항상 표시(19행). **♯행은 현재 마디에 그 음이 존재할 때만 표시**되며, 표시된 동안에는 일반 행과 동일하게 동작(빈 칸 탭 입력 가능). ♯음의 최초 생성은 음높이 스테퍼로만 가능. 마디에서 마지막 ♯음이 사라지면 행도 사라짐.
- 행 라벨: 계이름(도~시), 도 행에는 옥타브 병기(도3/도4/도5/도6). ♯행 라벨 볼드.
- 셀 탭 규칙:
  - 빈 칸 → **4분음표 고정 길이**로 입력(곡 끝 클램프) + 선택 + 프리뷰 사운드 + 겹침 해소.
  - 켜진 칸(다른 음) → 선택 + 프리뷰 사운드.
  - 선택된 칸 재탭 → 삭제.
- 쉼표 행(파생 표시): 어떤 음도 없는 칸이 점등. 탭 = "그 지점부터 쉼표 만들기" — 탭 지점이 음의 머리면 그 음 삭제, 중간이면 `d = 탭지점 - s`로 자르기. 빈 지점이면 토스트 "이미 쉼표입니다".
- 마디 경계를 넘는 음: 마디 마지막 칸에 ⤳ 표시.
- 스크롤: 선택 음이 보이도록 자동 스크롤(여유 60px), 그 외에는 스크롤 위치 보존. 초기 위치는 도5 부근.
- 재생 중 패드 플레이헤드(세로 레드 라인)가 현재 마디 구간에서 진행 표시.

### 3.6 마디 이동 바 (패드 바로 아래)
- `[◀] [마디 N / M] [▶]`.
- 마지막 마디에서 ▶ → 레드 **＋**: 누르면 일반 마디 1개 추가 후 이동.
- 첫 마디에서 ◀ → 레드 **＋**: 누르면 **못갖춘마디(2박, 8스텝) 추가**. 이때 모든 노트 `s+=8`, 코드 키 `+2`, mAcc 키 `+1`. 못갖춘마디가 이미 있으면 ◀ 비활성(opacity 0.3), 중복 추가 불가.

### 3.7 스테퍼 (드래그 없음, 클릭 전용)
- 좌: **음높이 세로 스테퍼** — [▲+다음 음 이름 / 현재 음(계이름+영문+옥타브) / ▼+이전 음 이름 / 캡션]. 반음 단위, 파3~도6 클램프. 변경 시 프리뷰 사운드(**소리 피드백은 음높이 변경·입력·선택에만. 위치/길이 변경은 무음**).
- 우 상: **위치 가로 스테퍼** — [◀ 앞으로 / "마디 N B.S" / 뒤로 ▶]. 16분 단위, 0~TOTAL 클램프, 이동 후 curM 추적 + 겹침 해소.
- 우 하: **길이 가로 스테퍼** — [◀ 짧게(이전 단계명) / 현재 길이명 / 길게(다음 단계명) ▶]. LEN_STEPS 단계 이동, 곡 끝 클램프, 겹침 해소.
- 선택 없으면 세 스테퍼 모두 비활성(opacity 0.35). 경계 도달 시 해당 방향 버튼 비활성.
- 길이 표기(SMP 금지): 16분/8분/점8분/4분/점4분/2분/점2분/온음표.

### 3.8 6버튼줄 (하단 고정)
| 버튼 | 동작 |
|---|---|
| ◀선택 | 이전 노트 선택+프리뷰. **선택이 첫 노트면 레드 ＋추가로 전환**: 첫 노트 앞에 같은 음높이 복제 삽입, `d=min(첫노트.d, 첫노트.s)`, `s=첫노트.s-d`. `s===0`이면 "곡 처음입니다" 차단. |
| ▶마디 (프라이머리) | 현재 마디만 재생 (`measStart(curM) ~ +measLen(curM)`). |
| 취소 | undo (스냅샷 스택, 최대 60). |
| 실행 | redo. |
| 삭제 (레드) | 선택 노트 삭제, 이전 노트 자동 선택. |
| 선택▶ | 다음 노트 선택+프리뷰. **선택이 마지막 노트면 레드 ＋추가로 전환**: 직전 음 복제를 뒤에 삽입(곡 끝 클램프, 곡 끝 도달 시 "마디를 추가하세요"). |

## 4. 화면 구성 — 열람 (`?mode=view`)

- 히어로: eyebrow "공유된 악보" + 제목 + 룰.
- 배지 4종: C장조 / 4/4 / ♩=템포 / N마디.
- 악보: **곡 전체 멀티라인**, 최대 높이 300px 세로 스크롤. 음영 없음. **음표 탭 = 그 위치부터 끝까지 재생.** 재생 중 플레이헤드 + 스크롤 추적.
- 큰 재생 버튼(전곡). **반주·메트로놈 미적용** (멜로디만).
- 하단: [링크 복사 | 복제해서 편집(편집 화면 전환)].

## 5. 도메인 알고리즘 (정확 구현 필수)

### 5.1 겹침 해소 (단선율)
변경/추가된 노트 ch의 구간 `[s, e)`와 겹치는 다른 노트:
- 앞에서 걸치면(`ns<s && ne>s`) `d = s-ns`로 자르고, 결과 d<1이면 제거.
- 그 외 겹침(시작이 구간 안)은 제거.

### 5.2 자동 쉼표 파생
마디별로 음이 점유하지 않는 구간 `[a,b)`를 순회하며:
```
c=a; while(c<b): take = [16,8,4,2,1] 중 첫 후보 cand s.t. (c % cand === 0) && (cand <= b-c) && (cand <= measLen)
```
take → 쉼표 종류: 16=온, 8=2분, 4=4분, 2=8분, 1=16분. (못갖춘마디에서 take 16은 8로 대체.)
결과 예: 빈 마디=온쉼표 1개, `[4,16)`=4분+2분, `[1,4)`=16분+8분.

### 5.3 코드 → 구성음 (반주 보이싱)
```
ROOT_PC = {C:0,D:2,E:4,F:5,G:7,A:9,B:11}  (♯:+1, ♭:−1)
ACC_INT = { "":[0,4,7], m:[0,3,7], 7:[0,4,7,10], maj7:[0,4,7,11],
            m7:[0,3,7,10], dim:[0,3,6], aug:[0,4,8], sus4:[0,5,7] }
root = 45 + ((pc - 9 + 12) % 12)          // 라2(45)~솔♯3(56)
tones = [root, root+iv[2], root+12+iv[1]] // 루트·5음(저) + 3음(옥타브 위) 오픈 보이싱
if (iv[3]) tones.push(root+12+iv[3])      // 7음도 옥타브 위
```
검증값: C→[48,55,64], Am7→[45,52,60,67], F♯dim→[54,60,69].

### 5.4 코드 지속 규칙
코드는 **다음 코드가 나올 때까지 지속** (마디 경계 무관, 리드시트 관례). `chordAtBeat(b)` = 키 ≤ b 인 최대 키의 코드.

### 5.5 반주 스케줄
박(4스텝) 단위로 순회하며 (코드, 유효패턴=마디 오버라이드||전역) 이 같은 연속 구간을 세그먼트로 그룹핑:
- **pad**: 세그먼트 시작에 전 구성음 1회 타건, 세그먼트 끝까지 지속. vol 0.075.
- **comp**: 박마다 전 구성음 2.3스텝 길이 타건. vol 0.07.
- **arp**: 8분음표(2스텝)마다 구성음을 순환(세그먼트 시작 시 인덱스 리셋), 1.9스텝 길이. vol 0.09.
- **off**(마디 오버라이드): 그 마디 구간 무음, 이후 재타건.
- **코드 없는 구간**(첫 코드 이전 등 `chordAtBeat(b)`가 없는 박): 반주 **완전 무음**, 코드 등장 시점부터 타건. (프로토타입 line 947 `if(!ch||pat==="off")` 동작과 동일)

## 6. 사운드 엔진

### 6.1 피아노 신스 (pianoAt)
- 트라이앵글 osc(f) + 사인 osc(2f, detune 3ct, gain 0.2) → gain → lowpass → 마스터.
- Lowpass: `min(f*7, 7500)`에서 `max(min(f*3,3200),400)`으로 음 길이에 걸쳐 지수 스윕, Q 0.3.
- **엔벨로프는 setTargetAtTime 기반 — 게인 불연속(순간 점프) 절대 금지** (팝 노이즈 원인):
  ```
  gain: setValueAtTime(0,t) → linearRamp(vol, t+0.005)
        → setTargetAtTime(vol*0.28, t+0.005, 0.3)   // 자연 감쇠
        → setTargetAtTime(0.0001, end, 0.045)        // 릴리즈
  osc stop = end + 0.35
  ```
- 마스터 체인: DynamicsCompressor(threshold −14, knee 18, ratio 5, attack 0.003, release 0.25) → gain 0.9 → destination.
- 생성된 osc는 activeNodes에 추적, 정지 시 일괄 stop.

### 6.2 재생
- 재생 시 범위 내 전 노트를 선행 스케줄(프로토타입 방식; 개선은 ARCHITECTURE 참조). 멜로디 vol 0.24, dur×0.96.
- 경과 스텝 `el = max(fromStep, (now-t0)/sps + fromStep)` — **시작 직후 음수 클램프 필수** (curM=-1 버그 방지).
- 재생 세션 `{ctx,t0,sps,fromStep,toStep,viewMode}`을 전역 보관 → 메트로놈/반주의 재생 중 토글이 이를 참조해 현재 위치 이후를 스케줄하거나 해당 노드만 정지.

### 6.3 메트로놈
- square osc, 액센트(각 일반 마디 1박) 1568Hz/vol 0.16, 일반 박 1047Hz/vol 0.1, 못갖춘마디 박은 전부 무액센트.
- 편집 화면 재생에만 적용. 열람 미적용.

### 6.4 소리 피드백 정책
프리뷰(0.35초, vol 0.22)는 **입력·선택·음높이 변경**에만. 위치/길이 변경, 마디 이동, 코드 변경은 무음.

## 7. URL 인코딩

```
hash = "v1." + base64url( JSON.stringify({
  t: title, q: tempo, m: meas, p: pickup,
  ap: accPat, am: mAcc, cb: 1,
  n: notes.map(n => [n.s, n.d, n.p]),
  c: chords
}))
```
- base64url: `btoa(unescape(encodeURIComponent(s)))` 후 `+→- /→_ =제거` (유니코드 제목 안전).
- 디코딩 하위 호환: `cb` 없으면 코드 키를 마디 키로 간주해 `×4`, 노트의 `"r"` 항목(구버전 쉼표)은 필터링, 누락 필드는 기본값.
- 공유 링크: `origin+pathname + "?mode=view" + "#" + hash`.

## 8. 상태·엣지 규칙 요약

- undo/redo: notes, chords, tempo, meas, pickup, accPat, mAcc 전체 스냅샷.
- 모든 파괴적 조작 직전 pushUndo (입력, 삭제, 이동, 길이, 코드, 마디/못갖춘 추가, 반주 설정 변경).
- restore 후 curM 범위 클램프, 사라진 선택 해제.
- 곡 끝/처음 경계는 토스트로 안내 후 무동작.
- 리사이즈 시 전체 재렌더(지오메트리 재계산).
