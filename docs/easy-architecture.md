# 쉽게 읽는 ScoreLink 구조 이야기

> 초등학생도 읽을 수 있게 쓴 설계 설명서예요.
> 정확한 어른용 설명은 `ARCHITECTURE.md`와 `superpowers/specs/`를 보세요.

## 1. 이 앱은 뭘 하는 앱이야?

**짧은 멜로디 악보를 만들어서, 링크 하나로 친구에게 보내는 앱**이에요.

1. 화면을 눌러서 음표를 콕콕 찍고
2. 재생 버튼으로 들어보고
3. "공유"를 누르면 링크가 복사돼요
4. 친구는 그 링크만 열면 악보가 보이고 소리도 나요. 회원가입도 필요 없어요!

## 2. 한 장 그림

```
        ┌───────────────────────────────┐
        │            ui (얼굴)           │  ← 버튼, 악보 그림, 피아노롤
        │  ┌─────────────────────────┐  │
        │  │     adapters (플러그)    │  │  ← 진짜 스피커·시계·주소창에 꽂는 부분
        │  │  ┌───────────────────┐  │  │
        │  │  │   ports (콘센트)   │  │  │  ← "이런 모양으로 꽂아줘" 약속
        │  │  │  ┌─────────────┐  │  │  │
        │  │  │  │engine(계획표)│  │  │  │  ← 언제 어떤 소리를 낼지 계산
        │  │  │  │ ┌─────────┐ │  │  │  │
        │  │  │  │ │core(두뇌)│ │  │  │  │  ← 음악 규칙과 계산만!
        │  │  │  │ └─────────┘ │  │  │  │
        │  │  │  └─────────────┘  │  │  │
        │  │  └───────────────────┘  │  │
        │  └─────────────────────────┘  │
        └───────────────────────────────┘
             화살표는 언제나 안쪽으로만! →→ 중심
```

양파처럼 겹겹이 되어 있어요. **바깥은 안쪽을 알아도 되지만, 안쪽은 바깥을 몰라요.**

## 3. 다섯 개의 방

### core — 두뇌 방 (src/core)
음악의 규칙이 전부 여기 있어요. "음표를 오른쪽으로 한 칸 옮기면 어떻게 될까?",
"이 마디의 시작은 몇 번째 칸일까?" 같은 **계산만** 해요.
화면도, 스피커도, 시계도 몰라요. 그래서 언제 어디서 실행해도 답이 똑같아요.

숫자에 **이름표**도 붙여요. 그냥 `7`이라고 하면 "7번째 칸"인지 "7번 음"인지
헷갈리잖아요? 그래서 `Step`(칸), `Midi`(음높이), `Sec`(초), `BarIndex`(마디)라는
이름표(branded type)를 붙여서 컴퓨터가 헷갈리면 혼내게 했어요.

### engine — 계획표 방 (src/engine)
"3번째 칸의 라(A) 음을 0.6초에 시작해서 0.9초에 끝내" 같은
**연주 계획표**를 만들어요. 여기도 계산만 하고, 진짜 소리는 안 내요.

### ports — 콘센트 방 (src/ports)
"스피커라면 이런 버튼이 있어야 해", "시계라면 지금 몇 시인지 알려줘야 해" 같은
**약속(인터페이스)** 만 적혀 있어요. 진짜 기계는 여기 없어요.

### adapters — 플러그 방 (src/adapters)
약속(콘센트)에 진짜 기계를 꽂아요.
- `web-audio-sink` → 진짜 브라우저 스피커
- `raf-clock` → 진짜 화면 시계
- `location-hash-store` → 진짜 주소창

그리고 **가짜 기계**(fakes)도 있어요! 가짜 스피커, 가짜 시계는 시험 볼 때 써요.
진짜 스피커는 소리가 잘 났는지 컴퓨터가 확인하기 어렵지만,
가짜 스피커는 "무슨 소리를 내라고 했는지" 전부 공책에 적어두거든요.

### ui — 얼굴 방 (src/ui)
버튼, 악보 그림, 피아노롤처럼 **눈에 보이는 것** 전부예요.
React로 만들었고, 색은 `styles/tokens.css`라는 물감 상자에서만 꺼내 써요.

## 4. 곡은 어디에 저장될까? — 주소 속에!

서버에 저장하지 않아요. 곡 전체를 꾹꾹 눌러 담아 **주소(URL) 뒤에 붙여요.**

```
https://scorelink.pages.dev/edit#v1.XQAAAA...  ← 이 뒷부분이 곡 전체!
```

편지 내용을 편지봉투 겉에 다 적어버리는 거예요. 그래서 링크만 보내면
친구 컴퓨터에서 곡이 다시 태어나요. 옛날 버전 링크도 열 수 있게 해뒀어요.

## 5. 쉼표의 비밀 — 저장하지 않아요

쉼표(쉬는 표시)는 데이터로 저장하지 않아요.
"음표가 없는 곳"을 보고 **자동으로 계산해서 그려요.**
물건을 두면 그림자가 저절로 생기는 것처럼요. 그래서 음표를 지우면
쉼표가 알아서 나타나고, 쉼표가 어긋나는 실수가 아예 생길 수 없어요.

## 6. 소리는 어떻게 내? — "뚝" 소리 금지!

브라우저의 Web Audio라는 스피커를 써요. 중요한 규칙이 하나 있어요:
**소리 크기를 갑자기 바꾸면 "뚝!" 하는 잡음이 나요.**
그래서 볼륨은 항상 미끄럼틀처럼 살살(setTargetAtTime) 줄여요.

## 7. 화면이 두 벌 — 같은 두뇌, 다른 옷

| | 휴대폰 (900px보다 좁으면) | PC (900px보다 넓으면) |
|---|---|---|
| 모습 | 하얀 배경, 한 마디씩 편집 | 어두운 DAW(작곡 프로그램) 모습 |
| 입력 | 드럼 패드처럼 콕콕 | 전곡 피아노롤에 콕콕 |
| 값 바꾸기 | 스테퍼 버튼 | 오른쪽 인스펙터 카드 |

문지기(`useMediaQuery`)가 화면 폭을 재서 옷을 골라 입혀요.
중요한 건 **두뇌(core)는 하나**라는 것! PC에서 음표를 찍어도, 폰에서 찍어도
똑같은 규칙(padTap)이 일해요. 옷만 다르지 사람은 같아요.

PC 다크 모드는 색깔 물감만 바꿔치기(`.daw` 토큰 재정의)해서,
악보 그리는 코드는 한 줄도 안 고치고 하얀 악보가 어두운 악보로 변신했어요.

## 8. 키보드 마법

PC에서는 키보드로도 편집할 수 있어요.

- `←→` 또는 `h l` : 이전/다음 음 고르기
- `↑↓` 또는 `k j` : 음을 높이/낮게
- `Shift`를 누르면 : 위치·길이 바꾸기
- `Space` 재생, `Enter` 한 마디 재생, `Delete` 지우기, `Cmd+Z` 되돌리기

**한글 자판이어도 돼요!** `ㅗㅓㅏㅣ`가 hjkl 자리인 걸 컴퓨터가 알아요.
글자 대신 **키의 자리**(e.code)를 보기 때문이에요.

## 9. 실수해도 괜찮아 — 되돌리기

뭔가 바꿀 때마다 바꾸기 **직전 모습을 사진**(스냅샷)으로 찍어둬요.
`Cmd+Z`를 누르면 사진첩에서 한 장 전 사진으로 돌아가요.

## 10. 시험은 어떻게 봐? — 가짜 기계와 정답지

- 코드를 고치기 전에 **시험 문제(테스트)부터** 만들어요. (TDD)
- 가짜 스피커·가짜 시계로 "정확히 이 소리를 이 시각에 내라고 했나?"를 검사해요.
- **골든 스냅샷**: 정답 소리 목록을 파일로 박제해두고, 코드를 고친 뒤에도
  소리가 그대로인지 비교해요. 답이 달라지면 "일부러 바꾼 게 맞아?"를 꼭 확인해요.
- 커밋 전에 4가지 검사를 모두 통과해야 해요:
  타입 검사 · 문법 검사 · 테스트(312개) · 금지 문자 검사

## 11. 친구들 광장 — 커뮤니티

로그인해서 곡을 게시하고 좋아요를 누르는 광장(community)도 있어요.
Supabase라는 바깥 창고를 쓰는데, **창고 열쇠(환경변수)가 없어도
편집기는 멀쩡히 열리게** 만들었어요. 열쇠가 필요한 순간에만 창고 문을 열고(지연 생성),
열쇠가 없으면 광장만 "준비 중"이 돼요. 예전엔 열쇠가 없으면 앱 전체가
하얀 화면이 됐는데, 그 버그를 고친 거예요.

## 12. 인터넷에 올리기 — 배포

`npm run build`로 앱을 상자(dist)에 담아서 Cloudflare Pages라는
무료 진열대에 올려요. 주소는 https://scorelink.pages.dev 예요.
곡이 주소 속에 저장되니까 서버 없이 정적 파일만으로 모든 게 돌아가요.

## 13. 딱 세 가지만 기억해요

1. **두뇌는 순수하게** — core는 계산만, 바깥세상은 몰라요
2. **곡은 주소 속에** — 링크가 곧 저장 파일이에요
3. **옷은 두 벌, 사람은 하나** — 폰과 PC 화면은 달라도 규칙은 같아요
